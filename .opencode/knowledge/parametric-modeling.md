## ЯДРО 3D-МОДЕЛИРОВАНИЯ

nanoCAD Механика PRO использует геометрическое ядро **C3D** (разработка АО «Топ Системы»).
Оно обеспечивает параметрическое 3D-моделирование, булевы операции, листовое тело и
сопряжения в сборках.

### Форматы файлов (Механика PRO)
| Тип | Расширение | Описание |
|-----|-----------|----------|
| Параметрическая деталь | *.dwp | Индивидуальный формат с историей 3D-построений |
| Сборка | *.dwa | Формат сборки со ссылками на .dwp-компоненты |
| Универсальный | *.dwg | Стандартный формат платформы nanoCAD |

### Режимы 3D-проектирования (Механика PRO)
1. **Параметрическая деталь** — полная история 3D-построений, управление эскизами
2. **Листовая деталь** — гибка, разгибы, сгибы, вырезы в тонколистовом металле
3. **Сборка** — вставка компонентов, наложение сопряжений (Mate, Insert, Angle, Tangent, Symmetry)
4. **Модель свободной формы** — direct editing без истории

## 1. ОБЩИЕ ПРИНЦИПЫ

Параметрическое моделирование — подход, при котором геометрия строится
на основе параметров (размеров, зависимостей, уравнений) и автоматически
перестраивается при их изменении.

### Терминология
- **Параметр** — числовая переменная (диаметр, длина, угол)
- **Зависимость** — связь между параметрами (A = B + 10)
- **Ограничение (Constraint)** — геометрическое условие (параллельность, концентричность)
- **Размер (Dimension)** — размерное ограничение (длина 100 мм)
- **Расчётный размер** — управляет геометрией; его изменение перестраивает модель
- **Справочный размер** — только информирует; редактирование не влияет на модель
- **Драйвер (Driver)** — параметр, управляющий другими
- **Подчинённый (Driven)** — параметр, производный от драйвера
- **Определённость эскиза** — все степени свободы зафиксированы (контур отображается чёрным)

### Этапы параметрического моделирования:

```
1. Определение параметров
   ↓
2. Создание параметрической геометрии (эскиз)
   ↓
3. Наложение геометрических ограничений
   ↓
4. Наложение размерных ограничений
   ↓
5. 3D-операция (Extrude/Revolve/Loft/Sweep)
   ↓
6. Проверка определённости (чёрный контур)
   ↓
7. Вариация параметров → автоматическая перестройка
   ↓
8. Фиксация и связь с чертежом (ассоциативность)
```

## 2. ТИПЫ ОГРАНИЧЕНИЙ (Constraints)

### 2.1. 2D-геометрические ограничения (эскиз)
В эскизе nanoCAD Механика PRO. Доступны только через GUI Механики (не через MCP),
но MCP-методы `nanocad_constraint_*` работают аналогично:

| Ограничение | MCP-инструмент | Описание |
|-------------|---------------|----------|
| Совпадение | `constraint_coincident(h1,h2)` | Стыковка точек отрезков/дуг |
| Коллинеарность | `constraint_collinear(h1,h2)` | Выравнивание осей |
| Концентричность | `constraint_concentric(h1,h2)` | Отверстия на одном центре |
| Параллельность | `constraint_parallel(h1,h2)` | Параллельные рёбра |
| Перпендикулярность | `constraint_perpendicular(h1,h2)` | Прямые углы |
| Горизонтальность | `constraint_horizontal(h)` | Выравнивание по оси X |
| Вертикальность | `constraint_vertical(h)` | Выравнивание по оси Y |
| Касательность | `constraint_tangent(line,curve)` | Сопряжение линии и дуги |
| Равенство | `constraint_equal(h1,h2)` | Равные длины/радиусы |
| Фиксация | `constraint_fix(h)` | Базовая точка (рекомендуется 1 на эскиз) |
| Симметрия | `constraint_symmetric(h1,h2,plane)` | Зеркальное отражение |
| Расстояние | `constraint_distance(h1,h2,d)` | Фиксация зазора |

**Важно (Механика PRO 2.5, 2026):**
- **Умное построение касательных** — система сама определяет точку соприкосновения
  и сразу накладывает геометрическую зависимость касательности
- **Умное зеркалирование** — зеркальное отражение автоматически проставляет
  зависимости между исходными и отражёнными объектами. Правки исходной геометрии
  сами применяются к отражению
- **Фаски и скругления с сохранением ограничений** — после обработки кромок все
  геометрические зависимости (параллельность, перпендикулярность) сохраняются

### 2.2. 3D-ограничения сборки (Assembly constraints)
В nanoCAD 3D Solid Modeling module и Механика PRO — 5 типов:

| Ограничение | MCP-инструмент | Степеней свободы | Описание |
|-------------|---------------|-----------------|----------|
| **Mate** (Совмещение) | `assembly_mate(h1, h2)` | 3 | Выравнивание граней/плоскостей |
| **Insert** (Вставка) | — (через отверстие) | 5 | Совмещение оси + плоскость — для крепежа |
| **Angle** (Угол) | `assembly_angle(h1, h2, a)` | 1 | Задание угла между элементами |
| **Tangent** (Касание) | `assembly_tangent(h1, h2)` | 1 | Касание цилиндрических/сферических поверхностей |
| **Symmetry** (Симметрия) | `assembly_symmetry(h1, h2, p)` | — | Симметрия относительно плоскости |

**Особенность Insert** (Механика PRO): ограничивает 5 из 6 степеней свободы —
совмещает ось отверстия и ось болта + плоскость прилегания. Используется для
автоматической сборки стандартного крепежа по ГОСТ/ISO.

### 2.3. Размерные ограничения (через DIM-инструменты)

| Инструмент | Назначение | Тип |
|-----------|-----------|-----|
| `create_linear_dimension` | Линейный размер (гориз./верт.) | Расчётный / Справочный |
| `create_aligned_dimension` | Наклонный размер | Расчётный / Справочный |
| `create_angular_dimension` | Угловой размер | Расчётный / Справочный |
| `create_radial_dimension` | Радиус | Расчётный / Справочный |
| `create_diametric_dimension` | Диаметр | Расчётный / Справочный |

**Прозрачный контроль размеров** (Механика PRO 2.5):
- Расчётные размеры управляют геометрией эскиза
- Справочные — только информируют, их изменение не влияет на параметрическую модель
- Визуально всегда видно, какой размер управляет чертежом, а какой просто информирует

## 3. СБОРКИ И СОПРЯЖЕНИЯ (Assemblies)

### 3.1. MCP-инструменты сборки
Доступны через `nanocad_assembly_*` (требуют .NET engine):

| Инструмент | Описание |
|-----------|----------|
| `insert_part(block_name, x, y, z)` | Вставка компонента-блока |
| `assembly_mate(h1, h2)` | Сопряжение Mate (совмещение) |
| `assembly_angle(h1, h2, angle)` | Угловое сопряжение |
| `assembly_symmetry(h1, h2, plane)` | Симметрия относительно плоскости |
| `assembly_tangent(h1, h2)` | Касание поверхностей |

### 3.2. Иерархия сборки (Механика PRO)
```
Сборка (*.dwa)
├── Деталь 1 (*.dwp)
│   ├── Эскиз (Sketch)
│   │   ├── Примитивы (линии, дуги, сплайны)
│   │   └── Ограничения (горизонтальность, совпадение, ...)
│   ├── Операция (Extrude, Revolve, Loft, Sweep)
│   ├── Элементы (Отверстие, Фаска, Скругление, Массив)
│   └── Параметры (переменные, допуски)
├── Деталь 2 (*.dwp)
│   └── ...
└── Сопряжения (Mate, Insert, Angle, Tangent, Symmetry)
```

### 3.3. Болтовые соединения (Механика PRO 2.5)
Автоматизированный инструмент для стандартного крепежа:

- Формирует одиночные болтовые соединения и массивы болтов, винтов, шпилек
  по ГОСТ и ISO
- Система автоматически подбирает длину крепежа
- Создаёт отверстия и формирует полный комплект: гайки, шайбы
- **Позиционирование по готовым отверстиям** (v2.5)
- **Поддержка листовых тел** (v2.5)
- **Импорт/экспорт 3D**: STEP, C3D, JT, IGES, Parasolid, ACIS, STL, Inventor,
  SOLIDWORKS, NX

### 3.4. Фиксация компонентов
- Фиксировать рекомендуется **только один базовый компонент** (например, «Основание»)
- Не следует фиксировать более одного тела или подсборки
- При переносе проекта используйте **относительные пути** и команду «Сформировать архив»

## 4. ПРИМЕР 1: ПАРАМЕТРИЧЕСКИЙ ВАЛ (MCP + revolve)

### 4.1. Определение параметров

```python
# Параметры вала (можно хранить в CAD-Spec YAML)
D1 = 30       # мм — диаметр первой ступени
D2 = 40       # мм — диаметр второй ступени
L1 = 100      # мм — длина первой ступени
L2 = 150      # мм — длина второй ступени
CHAMFER = 2   # мм — фаска 2×45°
KEY_W = 8     # мм — ширина шпоночного паза
```

### 4.2. Создание слоёв

```python
create_layer(name="Вал_Контур")
create_layer(name="Вал_Оси")
create_layer(name="Вал_Размеры")
set_current_layer(name="Вал_Оси")
```

### 4.3. Построение эскиза

```python
# Осевая линия
h_axis = create_line(x1=0, y1=0, x2=L1+L2+20, y2=0)

# Контур вала (половина сечения для вращения)
set_current_layer(name="Вал_Контур")
h_line1 = create_line(x1=0, y1=0, x2=0, y2=D1/2)
h_line2 = create_line(x1=0, y1=D1/2, x2=L1, y2=D1/2)
h_line3 = create_line(x1=L1, y1=D1/2, x2=L1, y2=D2/2)
h_line4 = create_line(x1=L1, y1=D2/2, x2=L1+L2, y2=D2/2)
h_line5 = create_line(x1=L1+L2, y1=D2/2, x2=L1+L2, y2=0)
```

### 4.4. Наложение геометрических ограничений

```python
# Совпадение точек стыков
constraint_coincident(h_line1, h_line2)
constraint_coincident(h_line2, h_line3)
constraint_coincident(h_line3, h_line4)
constraint_coincident(h_line4, h_line5)

# Горизонтальность/вертикальность
constraint_horizontal(h_line2)
constraint_horizontal(h_line4)
constraint_vertical(h_line1)
constraint_vertical(h_line3)
constraint_vertical(h_line5)
```

### 4.5. Размерные ограничения

```python
set_current_layer(name="Вал_Размеры")
create_linear_dimension(x1=0, y1=-10, x2=L1, y2=-10, dim_x=0, dim_y=-15, direction="horizontal")
create_linear_dimension(x1=L1, y1=-10, x2=L1+L2, y2=-10, dim_x=0, dim_y=-15, direction="horizontal")
create_diametric_dimension(center_x=L1/2, center_y=0, arc_x=L1/2, arc_y=D1/2)
create_diametric_dimension(center_x=L1+L2/2, center_y=0, arc_x=L1+L2/2, arc_y=D2/2)
```

### 4.6. 3D-операция (тело вращения)

```python
h_profile = create_polyline(
    vertices=[[0,0],[0,D1/2],[L1,D1/2],[L1,D2/2],[L1+L2,D2/2],[L1+L2,0]],
    closed=True)
h_shaft = revolve_solid(handle=h_profile, axis_x=0, axis_y=0, axis_z=1,
                         dir_x=1, dir_y=0, dir_z=0, angle=360)
```

### 4.7. Шпоночный паз (булево вычитание)

```python
h_key_box = create_box(x=KEY_W*3, y=D2*0.4, z=KEY_W)
move_solid(handle=h_key_box, dx=L1, dy=D2/2-D2*0.2, dz=0)
boolean_subtract(handle1=h_shaft, handle2=h_key_box)
```

## 6. ПРИМЕР 3: ПАРАМЕТРИЧЕСКАЯ ВИНТОВАЯ ПРУЖИНА

```python
# Параметры
D = 40      # Средний диаметр
d = 4       # Диаметр проволоки
n = 8       # Число витков
H0 = 60     # Высота в свободном состоянии

# Создание профиля (окружность)
h_circle = create_circle(cx=0, cy=0, radius=d/2)

# Создание пути (спираль — через сплайн)
# Точки: смещение по Z на шаг = H0/n
step = H0 / n
pts = []
for i in range(n * 4 + 1):
    angle = i * 360 / 4
    z = i * step / 4
    pts.append([D/2 * cos(radians(angle)), D/2 * sin(radians(angle)), z])
h_path = create_spline(fit_points=pts, degree=3, closed=False)

# Sweep — профиль по пути (.NET engine)
h_spring = sweep_solid(profile_handle=h_circle, path_handle=h_path)
```

## 8. BEST PRACTICES

### 8.1. Организация параметров

```
Правильные принципы:
✅ Один источник истины — параметр определён 1 раз
✅ Названия параметров — осмысленные (width, а не a1)
✅ Параметры сгруппированы по смыслу
✅ Допуски — отдельно от номиналов
✅ Имена переменных: буквы, цифры и _ (подчёркивание)
✅ Пространства имён через _ (например, _DIN_934, _Корпус_Толщина)

Ошибки:
❌ Магические числа в коде (hardcoded)
❌ Дублирование одного размера в разных местах
❌ Размеры, не связанные с базой
❌ Фиксация более одного компонента в сборке
```

### 8.2. Управление допусками (Механика PRO)
- Поля допусков управляются через таблицу допусков (выбор из ГОСТ)
- Переключение типа допуска (H7, k6, и т.д.)
- Допуски можно назначать на размерные ограничения в эскизе
- Загрузка переменных из внешнего файла, блоков и других документов

### 8.3. Порядок наложения ограничений

```
1. Сначала фиксация (Fix) — базовая точка/линия (только 1!)
2. Затем геометрические (Горизонтальность, Вертикальность)
3. Затем размерные (Линейные, Угловые) — расчётные
4. Справочные размеры — по желанию
5. В конце — зависимые (Равенство, Симметрия)
```

### 8.4. Массовые характеристики (Механика PRO 2.5)
Новый интерфейс сбора всех ключевых параметров в одном месте:

```python
# Через MCP (стандартный путь)
props = get_solid_properties(handle=h_shaft)
print(f"Объём: {props['volume']} мм³")
print(f"Площадь: {props['area']} мм²")
print(f"Центр масс: ({props['centroid_x']}, {props['centroid_y']}, {props['centroid_z']})")
```

### 8.5. Решаемые проблемы

| Симптом | Причина | Решение |
|---------|---------|---------|
| Не сходится эскиз | Наложено больше ограничений, чем степеней свободы | Удалить лишние |
| Геометрия «прыгает» | Не хватает ограничений относительно базы | Добавить Fix/Размер |
| Деталь не обновляется | Зависимость не замкнута | Проверить цепочку |
| Неверная сборка | Нет сопряжений или зафиксировано >1 компонента | Фикс: 1 база + Mate |
| Импорт не редактируется | STEP/IGES — непараметрическое тело | Создать новые параметрические элементы поверх |
| Пропали ссылки | Файлы перемещены | Использовать «Сформировать архив» + относительные пути |

### 8.6. Инструменты проверки параметризации

```python
# После построения — проверить геометрические свойства
info = get_solid_properties(handle=h_shaft)
print(f"Объём: {info['volume']} мм³")

# Проверить расстояние между объектами
dist = get_distance(x1=10, y1=0, x2=100, y2=0)
print(f"Расстояние: {dist} мм")

# Проверить площадь
area = get_area(handle=h_face)
print(f"Площадь: {area} мм²")
```

## 9. ПАРАМЕТРИЧЕСКАЯ МОДЕЛЬ КРЕСЛА (HT-Chair)

### 8.1. Параметры модели

| Параметр | Значение | Описание |
|----------|----------|----------|
| W | 620 мм | Ширина кресла |
| D | 450 мм | Глубина сиденья |
| H | 850 мм | Общая высота |
| T | 5 мм | Толщина пластика |
| TW | 40 мм | Ширина трубы |
| TH | 20 мм | Высота трубы |
| WT | 2 мм | Толщина стенки трубы |

### 8.2. Зависимости

```
H_seat = 450 мм
H_armrest = 550..690 мм
θ_backrest = 12°
D_backrest = 520 мм (глубина основания спинки)
W_frame = W = 620 мм
L_tube = 2 × (260 + 450 + 190 + 450 + 80) + 620 + 620 + 190 = 4290 мм
V_steel = L_tube × (TW×TH - (TW-2×WT)×(TH-2×WT)) ≈ 960 000 мм³
M_steel = V_steel × 7.85e-6 ≈ 7.5 кг
```

### 8.3. Структура сборки

```

├── Frame (Сварной каркас, Ст3сп)
│   ├── LeftSide (5 сегментов 40×20×2)
│   ├── RightSide (5 сегментов, отражение)
│   └── Crossbars (3 поперечины)
├── Seat (Сиденье, ABS 5 мм)

├── Armrest_Left (Подлокотник левый)

```

## 10. ИНТЕГРАЦИЯ С БАЗОЙ ЗНАНИЙ

При параметрическом расчёте используйте:

- **materials.md** — для выбора материала и его свойств
- **tolerances.md** — для назначения допусков (IT-квалитеты)
- **mechanics.md** — для прочностных расчётов
- **engineering-standards.md** — для стандартных элементов
- **cad-model-format.md** — для оформления CAD-Spec YAML

### Типовой цикл параметрического проектирования:

```
1. Запросить CAD-Designer → CAD-Spec YAML
2. Прочитать секции: params, entities, tolerances
3. Выбрать материал (materials.md)
4. Рассчитать сечения (mechanics.md)
5. Назначить допуски (tolerances.md)
6. Построить параметрическую модель (MCP инструменты)
7. Наложить ограничения (constraints)
8. Проверить (вариация параметров)
9. Экспортировать (DWG, PDF, STL)
```

## 11. ОГРАНИЧЕНИЯ И ОСОБЕННОСТИ

- Геометрические ограничения (constraint_*) требуют **.NET engine**
- Размерные ограничения (create_linear_dimension и др.) — **базовые методы nanoCAD**
- Сборки (assembly_*) — только через .NET engine
- align_3d может изменять локальную СК объекта — позиционирование лучше делать через move_solid
- rotate_entity вращает вокруг глобальной оси Z — для поворота вокруг произвольной оси используйте align_3d
- create_box создаёт тело с центром в начале СК — всегда используйте move_solid после создания
- boolean_union/subtract поглощают handle2 в handle1 — handle2 удаляется из чертежа
